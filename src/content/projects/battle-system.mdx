This isn't trying to ship. It's a sketchbook for one question: how do you model
turn-based combat so that adding a new status effect doesn't require touching
ten files?

Every rewrite chases a different answer. The current one models the whole battle
as an event queue you can replay and inspect, which is elegant until you need a
reaction that fires *during* another action's resolution. That's the wall the
fourth rewrite will start at.

Marked broken on purpose. It does what I need — it teaches me something each time
I take it apart.
