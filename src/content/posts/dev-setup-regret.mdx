A short one. Patch notes for my own machine, written mostly so future-me knows
who to blame.

## + moved to a single kitty config with includes

I split my terminal config into a base file and per-machine overrides stitched
together with `include` directives. Cleaner on paper. In practice I now have to
remember which of four files a given setting lives in. Net neutral, leaning
negative.

## + replaced three plugins with twelve lines of lua

I had three Neovim plugins doing what turned out to be a small amount of actual
work. I deleted them and wrote the behavior myself:

```lua
vim.api.nvim_create_autocmd("BufWritePre", {
  pattern = "*",
  callback = function()
    local save = vim.fn.winsaveview()
    vim.cmd([[keeppatterns %s/\s\+$//e]])
    vim.fn.winrestview(save)
  end,
})
```

This trims trailing whitespace on save without yanking my cursor around. It is
the single most satisfying thing in the config and I will defend it.

## + one config file that has grown teeth

My shell init has quietly become a 400-line organism. It sources other files,
defines functions that define functions, and has at least one block I'm afraid
to touch because I no longer remember what depends on it. This is the regret in
the title. I know it's coming. I'm choosing not to deal with it today.

## - removed the fancy prompt

The prompt that showed git status, node version, k8s context, the phase of the
moon — gone. It added latency I could *feel* on every `cd`. I replaced it with a
path and a `$`. I do not miss the moon.

That's the patch. No major version bump. Ship it and find out.
